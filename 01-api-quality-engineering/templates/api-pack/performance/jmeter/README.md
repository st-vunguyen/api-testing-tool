# JMeter Performance Pack

This folder extends the base performance starter with a more standard JMeter-oriented stack.

Use it when you need one or more of these:

- a standard `.jmx` test plan handoff
- JMeter HTML dashboard output
- richer thread-group / timer / plugin options
- a conversion path from API collections or workload docs into JMeter assets
- an executive HTML dashboard derived from standard JMeter report data

## Included files

| File | Purpose |
|---|---|
| `starter-test-plan.jmx` | Safe starter JMeter plan with placeholders |
| `user.properties` | User-level knobs for CLI overrides |
| `reportgenerator.properties` | Dashboard generation defaults |
| `plugins-catalog.md` | Recommended plugin set and when to use each |
| `run-jmeter.sh` | macOS/Linux runner |
| `run-jmeter.ps1` | Windows runner |
| `render-jmeter-dashboard.js` | Generates an executive-style HTML dashboard from JMeter stats |
| `datasets/README.md` | Expected dataset layout and safety notes |

## Suggested flow

1. Start with prompt `23-jmeter-stack-setup` to validate install, plugins, and folder structure.
2. Use prompt `24-jmeter-convert-collections` to translate workload intent into a real `.jmx`.
3. Use prompt `25-jmeter-execute-and-report` to run the plan and generate standard raw outputs.
4. Use prompt `26-jmeter-report-analysis` to turn raw JMeter artifacts into a curated report and dashboard.

## Standard raw output target

Store JMeter raw outputs under:

- `tools/api-quality-engineering/<slug>/reports/raw/performance/jmeter/<run-slug>/`

Recommended contents:

- `results.csv`
- `jmeter.log`
- `report-output/` (standard JMeter HTML dashboard)
- `report-output/statistics.json` when available
- `dashboard.html` generated by `render-jmeter-dashboard.js`

## Safety defaults

- Keep `SAFE_TO_RUN=no` until the environment is approved.
- Prefer dedicated `local`, `dev`, or `perf` targets.
- Treat plugin-heavy thread groups as opt-in, not default.
- Never assume thresholds; read them from the strategy or report assumptions.
